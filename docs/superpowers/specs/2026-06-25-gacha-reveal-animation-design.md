# 金色キャラ ガチャ演出 設計仕様

作成日: 2026-06-25

## 目的

役職ガチャの当選演出を、REALITY のガチャ風に差し替える。「回す」ボタンを押すと画面が暗転し、金色のキャラクターが上から降ってきて拡大・キラキラした後、中央に結果カード（役職名＋カクテル言葉）を表示する。

## スコープ

- 既存のガチャ機イラストと「回す」ボタンは**残す**。
- 差し替えるのは「回す」押下後の演出（現状: カプセル落下 → 開封 → 紙吹雪付き結果表示）のみ。
- 通常画面の背景はピンクのまま。演出中だけ暗転オーバーレイを表示し、終了後（「もう一回」）に通常画面へ戻る。

非スコープ（YAGNI）:
- 効果音は付けない。
- ガチャ機イラスト自体の変更や、複数キャラ・レアリティ分岐は行わない。

## 演出フロー

「回す」押下後、暗転オーバーレイ上で以下を順に再生する。

| 段階 | 内容 | 目安時間 |
|------|------|----------|
| 1. 暗転 | 全画面に暗いオーバーレイをフェードイン | 〜0.3s |
| 2. 降下 | 金色キャラが画面上部から落下し、中央でバウンドして静止 | 約0.7s |
| 3. 2回ふくらむ | 1.7倍に拡大 → 元サイズ を2回。ゆっくりめ | 約4.4s |
| 4. キラキラ | キャラ周囲に光の粒が弾ける | 段階3後半〜 |
| 5. 結果カード | キャラが縮小フェードアウトし、中央に結果カードがポップイン | 約0.5s |
| 6. 操作 | 既存「保存」「もう一回」ボタンを表示。「もう一回」で通常画面へ | - |

合計 約6秒。降下中・拡大中はテキストを出さず、最後に結果カードのみ表示する。

## 結果カードの構成

- 役職名（例: 「まじめなマンハッタン ✨」）— ピンクの太字見出し
- 白背景のカード内に:
  - 🍸 カクテル言葉：「{meaning}」
  - ひとこと：{note}
  - 材料：{ingredients を ` / ` 区切り}

内容は既存 `ResultDisplay` が表示している `title` / `info`（meaning, note, ingredients）をそのまま使う。

## コンポーネント設計

### App.jsx（フェーズ管理）
- フェーズを `idle → revealing → revealed` に整理する。
  - `revealing`: 暗転オーバーレイ＋ `GachaReveal` 再生中。
  - `revealed`: 結果カード＋保存／もう一回を表示（暗転オーバーレイは継続）。
- `handleTurn`: `drawTitle()` / `pickCapsuleColor()` を呼び、`revealing` へ。アニメ完了（タイマー）で `revealed` へ。
- `handleReset`: `idle` に戻し、通常のピンク画面へ。
- 既存タイマー管理（`timers` ref / `clearTimers`）の仕組みを踏襲する。

### GachaReveal.jsx（新規）
- 役割: 暗転オーバーレイ上で金色キャラの降下・2回拡大・キラキラを再生する。
- props: `image`（キャラ画像）、`onComplete`（任意。アニメ終了通知）。
- 実装: CSS keyframes（`drop` / `pulse` / `sparkle`）。キラキラの粒は配列で描画。
- アニメ尺は単一の定数（例 `REVEAL_MS`）で管理し、App.jsx のタイマーと一致させる。

### ResultDisplay.jsx（改修）
- 紙吹雪（confetti）を削除し、暗転オーバーレイ上のカード表示用にスタイル調整。
- 表示内容（役職名・カクテル言葉・ひとこと・材料）のロジックは維持。

### Capsule.jsx
- 演出フローから外す（App.jsx から参照を削除）。コンポーネントファイルは削除する。

### 画像アセット
- `~/Downloads/ChatGPT Image 2026年6月23日 23_28_20.png`（金色のキャラ、透過PNG）を軽量化のためリサイズし `src/assets/` に配置して import する。

## アクセシビリティ

- `prefers-reduced-motion: reduce` の場合は降下・拡大アニメを抑制し、結果カードを（ほぼ）即時表示する。

## テスト方針

- `App` のフェーズ遷移（idle → revealing → revealed → reset で idle）を検証。タイマーはフェイクタイマーで進める。
- `GachaReveal` が渡された `image` を表示し、所定時間後に `onComplete` を呼ぶこと。
- `ResultDisplay` が `title` と `info`（meaning / note / ingredients）を正しく描画すること（既存テストを踏襲・更新）。
- 既存の `draw.js` 等のテストは変更しない。

## 確認済みの意思決定

- 演出範囲: ガチャ機は残し、押下後の演出のみ差し替え。
- 拡大: 1.7倍を2回、ゆっくり（プレビュー v2 の速度）。
- 結果表示時: キャラは縮小フェードアウトで消す。
- 背景: 通常はピンク、押下時に暗転オーバーレイ。
