# 役職（カクテル）重複排除 設計

## 背景・目的

現在の `drawTitle()` は形容詞・カクテルともに完全ランダムに抽選するため、
既に別の人に割り当て済みのカクテル（＝役職）が再び出てしまうことがある。
カクテルは全50種類で、それぞれ `cocktailInfo` に固有の意味・ネタが紐づく
「役職」そのものなので、同じ会・同じDBの中で同じカクテルが2人に割り当たるのは
避けたい。形容詞は演出上の装飾でしかないため対象外とする。

## 判定範囲

- 重複判定は **カクテル名のみ**（形容詞は無関係、同じカクテルでも形容詞が違えば
  別の見た目にはなるが、役職としては同一とみなし除外する）。
- 判定対象は **DB全体に保存済みの人（`people` テーブル）** に紐づくカクテル。
  ブラウザのリロードや別端末からの利用をまたいで有効。

## データソース

新規のサーバー・DB変更は行わない。既存の `GET /api/people`
（`src/lib/api.js` の `fetchPeople`、`server/index.js` の該当ハンドラ）が
保存済み全員の `cocktail` フィールドを含む配列を返すので、これをそのまま使う。
クライアント側で `new Set(people.map(p => p.cocktail))` として使用済み集合を作る。

## クライアントフロー

1. `App.jsx` で `view` が `'gacha'` になったとき（`handleSelectGacha` 実行時）に
   `fetchPeople()` を呼び、使用済みカクテル集合 `usedCocktails` を state に持つ。
   取得中は抽選ボタンを無効化し、ロード中である旨を表示する。
   取得に失敗した場合はエラーメッセージを表示し、ボタンは無効のままにする
   （このガチャ画面を出て入り直すことで再取得を試みる、という既存の簡素な
   エラーハンドリング方針に合わせる）。
2. `drawTitle` (`src/lib/draw.js`) のシグネチャを
   `drawTitle(excludeCocktails = [])` に変更する。
   - 形容詞は従来通り全体からランダムに1つ選ぶ。
   - カクテルは `cocktails` から `excludeCocktails` を除いた残りからランダムに選ぶ。
   - 残りが0件（＝全50種類使用済み）の場合は `null` を返す。
3. `handleTurn`（`App.jsx`）は抽選前に残りカクテル数が0かどうかを確認する。
   0件であれば「役職はすべて割り当て済みです」というメッセージを表示し、
   抽選ボタンはあらかじめ無効化しておく（`GachaMachine` の既存 `disabled` prop
   を流用）。
4. 保存（`SaveResult` の `onSave` 経由で `saveResult` API 呼び出し）が成功したら、
   `App.jsx` は再フェッチせず、ローカルの `usedCocktails` に
   その場でドロー結果の `cocktail` を追加する（同一セッション内での即時反映）。

## UI

- ガチャ画面（`view === 'gacha'` の `sub-view`）に、ロード中／エラー／
  枯渇のいずれかの状態メッセージを `GachaMachine` の付近に表示する。
  既存の `GeneratePage` のエラー表示（`.generate-page__error` 相当の
  段落要素）に倣った簡素な文言表示とし、新規コンポーネントは作らない。
- 枯渇時・ロード中・エラー時は `GachaMachine` の `disabled` を `true` にする。

## 変更対象ファイル

- `src/lib/draw.js` — `drawTitle` のシグネチャ変更、除外ロジック、枯渇時 `null` 返却。
- `src/lib/draw.test.js` — 除外指定時に除外カクテルが選ばれないこと、
  全除外時に `null` を返すこと、形容詞は除外の影響を受けないことのテストを追加。
- `src/App.jsx` — `usedCocktails` state、`fetchPeople` 呼び出し、
  ロード／エラー／枯渇の状態管理、`handleTurn` の抽選前チェック、
  保存成功時のローカル追加処理、状態メッセージの表示。

## スコープ外（やらないこと）

- サーバー・DBスキーマの変更、新規APIエンドポイントの追加。
- 形容詞側の重複排除。
- 複数ガチャ（gachas.js に複数エントリがある場合）をまたいだ排他制御
  （現状ガチャは `cocktail` の1種類のみのため考慮しない）。
- 一定時間ごとのポーリングによる自動再取得。

## テスト観点

- `drawTitle([])` は従来通り全カクテルから選ばれる。
- `drawTitle(excludeList)` は `excludeList` に含まれるカクテルを返さない。
- 全カクテルを除外リストに渡すと `drawTitle` は `null` を返す。
- 除外リストを渡しても形容詞の抽選範囲は変わらない。
- （手動確認）ガチャ画面に入ると使用済みカクテルを考慮した抽選になり、
  保存後は同一セッション内でそのカクテルが再抽選されない。
- （手動確認）全50種類使用済みの状態でガチャ画面に入るとボタンが無効化され、
  枯渇メッセージが表示される。
