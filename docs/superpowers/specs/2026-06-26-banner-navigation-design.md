# バナー遷移ナビゲーション 設計

## 背景・目的

現在の `App.jsx` は画面上部に「一覧 / ガチャ / 生成」の3タブ (`.view-nav`) を
**常時表示**し、どの画面でもタブで切り替える構成になっている。

これを、よくあるアプリの遷移 — **バナーをクリックして詳細へ進み、「一覧に戻る」で戻る**
— に変更する。常時タブを廃止し、各画面が自分の遷移手段を持つ。

## 方針

ルーターは導入しない。既存の `view` ステート方式
(`'list' | 'gacha' | 'generate'`) はそのまま活かす。
変更点は「常時タブナビの廃止」と「各画面への戻る手段の追加」のみ。最小変更で実現する。

## 画面構成

### ① 一覧画面 (`view === 'list'`)
- バナーを並べる。バナークリックで `handleSelectGacha` を呼びガチャ画面へ遷移（実装済み）。
- 一覧の末尾に**小さな「生成」入口**（控えめなテキストリンク/小ボタン）を置き、
  クリックで `setView('generate')`。普段は目立たせない（管理者向け）。

### ② ガチャ画面 (`view === 'gacha'`)
- 画面上部左に **「← 一覧に戻る」ボタン**を表示。
  クリックで `handleReset()` を呼んでから `setView('list')`。
- 既存のガチャ機・演出・結果表示は変更しない。

### ③ 生成画面 (`view === 'generate'`)
- 画面上部左に **「← 一覧に戻る」ボタン**。クリックで `setView('list')`。
- 既存の GeneratePage は変更しない。

## 共通部品

「← 一覧に戻る」は2画面で使うため、小さな共通要素（`BackButton` または
共通クラス付き要素）として用意する。`onClick` で `view='list'` に戻す。
ガチャ画面では戻る前に `handleReset()` も呼ぶ。

## 削除・整理するもの

- `App.jsx` の `<nav className="view-nav">…</nav>` ブロック（3タブ）を丸ごと削除。
- `App.css` の `.view-nav` / `.view-nav__btn` スタイルを削除。

## スコープ外（やらないこと）

- ブラウザ戻るボタン対応・URLハッシュ連動。
- 演出ロジック、保存・生成API、gacha データ構造の変更。

## テスト観点

- 一覧でバナークリック → ガチャ画面に遷移する。
- ガチャ画面の戻るボタン → 一覧に戻り、phase が idle にリセットされる。
- 一覧の生成入口 → 生成画面に遷移する。
- 生成画面の戻るボタン → 一覧に戻る。
- 常時タブ (`.view-nav`) が表示されないこと。
