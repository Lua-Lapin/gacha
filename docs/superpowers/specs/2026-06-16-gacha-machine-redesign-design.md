# ガチャ本体デザイン リニューアル 設計書

作成日: 2026-06-16

## 概要

役職ガチャの本体（筐体）ビジュアルを、クラシックなカプセルトイ（ガムボール
マシン）風のリファレンス画像に寄せてリニューアルする。現状の `<div>` 組み合わせ
による簡易描画を、ディテールを忠実に再現できるインライン SVG に置き換える。

抽選ロジック・状態フロー・データは変更しない。今回はあくまで `GachaMachine` の
見た目のリデザインに限定する。

## 目的・成功基準

- リファレンス画像の質感（銀のクロームダイヤル・二色カプセル・グレー台座）が
  一目で伝わる本体になっている
- 既存の演出フロー（idle / spinning / dropping / revealed）がそのまま動く
  - `shaking` 中はドームが揺れる
  - 落下カプセル `Capsule` と結果表示 `ResultDisplay` が本体の上に正しく重なる
- 「回す」操作がダイヤルから直感的に行える（キーボード／スクリーンリーダー対応含む）

## スコープ

### やること
- `GachaMachine` を `<div>` 群から 1 枚のインライン SVG イラストへ置き換え
- 取り入れる画像要素（確定済み）:
  - 二色カプセル（白＋斜め半分カラー）をドーム内に固定配置
  - 中央のシルバー・クロームダイヤル（十字スリット）＋コイン投入口
  - 赤い筐体の下のグレー台座＋短い脚
- ダイヤル自体を「回す」ボタンにする（下に小さく「回す」キャプション）
- 揺れ／ダイヤル回転のアニメーションを CSS で整理

### やらないこと（YAGNI）
- 星・キラキラなどの周辺装飾（画像にはあるが今回は不採用）
- 抽選ロジック・確率・データ・状態フローの変更
- `Capsule` / `ResultDisplay` の演出ロジック変更（重なりが崩れない範囲の調整は可）

## デザイン詳細

### 描画方式
- `GachaMachine` の返り値を 1 枚のインライン SVG にする
- 物理配色のシーンのため、SVG 内の色はハードコード hex を使い、ダークモードでも
  反転させない（テーマ用クラスは使わない）
- ドーム内カプセルの色は既存 `CAPSULE_COLORS`（`src/lib/draw.js`）を流用する

### 各パーツ
- **ガラスドーム**: 水色ガラス＋上部ハイライト＋青いリム。中に二色カプセルを
  6 個固定配置（演出用の飾り）。`shaking` 時はドーム部分が揺れる。
- **二色カプセル**: 白い円＋斜め半分にカラー帯。各カプセルに `CAPSULE_COLORS`
  から色を割り当てる。
- **クロームダイヤル**: 中央のシルバー丸ダイヤル＋十字スリット。これ自体が
  クリック可能なボタンで、`onTurn` を発火し、`disabled` と
  `aria-label="ガチャを回す"` を持つ。ホバー／クリックでダイヤルが回転する演出。
  すぐ下に小さく「回す」キャプションを置く。
- **コイン投入口**: ダイヤル左の黒い細長スリット（装飾）。
- **筐体下部**: 赤い本体に黒い取り出し口。その下にグレーの台座と短い脚 2 本。

### 操作（回す）
- ダイヤルが `<button>`（または `role="button"` を持つフォーカス可能な SVG 要素）
  として機能する
- `onTurn` / `disabled` の props 契約は現状を踏襲する
- アクセシビリティ: キーボードフォーカス可能・`aria-label` 付与・Enter/Space で発火

## コンポーネント設計（変更範囲）

- `App`: 変更なし（状態・抽選・文字列管理はそのまま）
- `GachaMachine`: SVG ベースに全面書き換え。props 契約（`shaking` / `onTurn` /
  `disabled`）は維持。
- `GachaMachine.css`: 揺れ・ダイヤル回転・ホバー等の keyframes / transition を整理。
- `Capsule` / `ResultDisplay`: 原則変更なし。本体差し替えで重なり位置がずれる
  場合のみ座標・サイズを微調整。

## 実装の進め方（SVG の作り込み）

SVG の細部（形状・配色・つや・各パーツの位置バランス）は、本設計書の方針を満たす
たたき台をまず実装したうえで、**dev サーバーのプレビューで実物を見ながら反復改善
する**。最終的な見た目はプレビュー上での確認で確定させる。

## テスト方針

- 抽選ロジックの既存単体テスト（`src/lib/draw.test.js`）は影響を受けない想定。
  リファクタで参照が変わらないことを確認する。
- 本体の見た目・演出・重なり・ダイヤル操作はブラウザ（プレビュー）で手動確認する。
  - idle / spinning / dropping / revealed の各状態で崩れがないこと
  - ダイヤルクリック・キーボード操作で `onTurn` が発火すること
