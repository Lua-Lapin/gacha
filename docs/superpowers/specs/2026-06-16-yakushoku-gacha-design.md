# 役職ガチャ（形容詞 × カクテル）設計書

作成日: 2026-06-16

## 概要

「役職ガチャ」を回すと、形容詞とカクテル名がランダムに組み合わさった肩書き
（例:「やかましいマティーニ」）が表示される、React + Vite 製の単一ページアプリ。
クラシックなカプセルトイ（ガチャガチャ）風の見た目で、ノブを回すとカプセルが落下・
オープンして結果が出る演出を持つ。

## 目的・成功基準

- 一目で「ガチャ」とわかるデザインで回せる
- 回すと「形容詞 + カクテル名」の文字列が表示される（完全ランダム、毎回独立）
- バックエンド不要、ブラウザだけで完結

## スコープ

### やること
- React + Vite のフロントエンドのみ
- 形容詞50個・カクテル50個のデータ保持
- カプセル落下式の回す演出
- 結果のちょい盛り演出（カプセル色ランダム + 確定時のキラッ + 軽い紙吹雪）
- 「もう一回」での再抽選

### やらないこと（YAGNI）
- 結果の仕込み／リグ機能（完全ランダムのみ）
- レア度・確率調整
- 履歴保存・ログイン・サーバー通信
- 重複防止（毎回独立ランダムでよい）

## 技術スタック

- React + Vite
- スタイル: CSS（keyframes / transition 中心の軽量アニメーション）
- 状態管理: React の `useState`（外部ライブラリ不要）

## データ

- `src/data/words.js`
  - `adjectives`: 形容詞・形容フレーズ 50個（配列）
  - `cocktails`: カクテル名 50個（配列）
- 抽選: `adjectives` と `cocktails` からそれぞれ `Math.random()` で1要素選び、
  `形容詞 + カクテル名` を結合

形容詞・カクテルの内容は本ブレストで確定済みの各50個を使用する。

## 画面・演出フロー（クラシック・カプセルトイ / 落下式）

1. **idle**: 赤い筐体＋透明ドーム＋カラフルなカプセル、中央に「回す」ノブ
2. **spinning**: ノブを回す（クリック）→ ドーム内のカプセルが揺れる
3. **dropping**: カプセルが1個コロンと落下口へ落ちる（カプセル色はランダム）
4. **revealed**: カプセルがパカッと開く → 肩書きをドンと表示。
   確定時にキラッと光る + 軽い紙吹雪
5. 「もう一回」ボタンで idle に戻り再抽選

## コンポーネント設計

- `App`
  - 抽選ロジック（ランダム選択）とアプリ状態管理
  - 状態: `idle` / `spinning` / `dropping` / `revealed`
  - 確定した肩書き文字列とカプセル色を保持
- `GachaMachine`
  - 筐体・ドーム・ノブの見た目と「回す」操作の受け口
  - ドーム内カプセルの揺れ表現
- `Capsule`
  - 落下＆オープンのアニメーション
  - props でカプセル色を受け取る
- `ResultDisplay`
  - 確定した肩書きの表示 + 紙吹雪・キラッの演出
- 演出の責務は各コンポーネントの CSS に閉じ、`App` は状態と文字列のみを管理する

## エラーハンドリング

- 外部依存・通信がないため失敗系はほぼ無い
- データ配列が空の場合に備え、抽選関数は配列長を前提にする（データは固定で常に50個）

## テスト方針

- 抽選ロジック（`形容詞 + カクテル名` を組み立てる純粋関数）を単体テストで検証
  - 返り値が `adjectives` の要素 + `cocktails` の要素の結合になっていること
- 演出・見た目は手動確認（ブラウザ）で担保
